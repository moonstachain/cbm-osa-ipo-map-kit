# 从零构建一张「战略地图-原子卡片」· 7 步配方（公开脱敏版）

给任意 IP / 业务 / 领域，按此 7 步产出一张 CBM-OSA-IPO 战略地图。
全程守诚实纪律：无金卡不虚报，🟢引真据 / 🟡推断 / ⚪公式默认。
（在本 kit 里：步骤 1-4 = 写弹夹 `data/maps/<id>/map.py`；步骤 5-7 = 打分器与渲染器代劳。）

## 前置 · 立信息源清册（IPO 的 I 地基）

先把目标的全要素信息源列成清册：每源记 **位置 / 口径 / 刷新频率 / 隐私分级 / 一手度 / 入仓状态**。
**清册的完备度 = 卡的 I 层"全"上限**。缺一源即 I 层"全"降级——诚实标出，不假装采全了。
PII 分层：明细留本机、出仓只聚合。

## 步骤 1 · 横向到边（切 MECE 领域）

把业务切成不重不漏的子领域，5-8 个为宜。两两问"是不是在测同一件事"（不正交则合并）
+"有没有整块漏了"（不全则补）。**空白领域也要显式立格**，标 candidates，盲区不静默。

## 步骤 2 · 纵向分层（每领域切三责任层）

每子领域切 Direct 定方向 / Control 控系统 / Execute 做动作 三层。得到 N领域 × 3层 的棋盘。
（弹夹里：每域三层都要有格，空白也立——模板 selfcheck 会查。）

## 步骤 3 · 每格一个 OSA（把这一格想透）

逐格填 O 目标 / S 策略 / A 行动判断句。本体锚：O=经信息五层与约束比较形成的全局最优目标
（≥3 候选）/ S=实现 O 的策略组合与科学实验 / A=SMART 行动合同。三段不跳层。

## 步骤 4 · 为每张 OSA 建一套统一 IPO 信息底座

在顶层 `information_engine` 登记全/真·一手/细的来源证据，按 L1→L5 保存连续加工产物，
让 Output 指向 Objective、Strategies、Actions，并记录现实结果如何成为下一轮 Input。
O/S/A 共享这一套底座，不复制平行 I/P/O 子树。

## 步骤 5 · IPO 与 OSA 分开评分、联合验收

- **I 采集**：全(不漏) / 真·一手 / 细，逐源判、引清册真源。
- **P 处理**：信息五层逐层留痕，L5 还须有方法引用与全局最优依据。
- **O 输出**：闭环 / 自动化 / 智能化分别绑定现实回执；回流不是第四个正式字母。
- **OSA**：O 按目标合同、S 按三阶段实验、A 按 SMART 与 A0–A3 独立计算。

机器判不了的覆盖完整性和全局最优必须有人审证据。Input 不可信时，所有 OSA `effective`
保持 `unassessed`；Output/回流不完整时不得判 Gold。

## 步骤 6 · 四门 + H/V/A/E 复验

四道门：究竟(追第一性) / 完备(MECE不重漏) / 递进(不跳层) / 原子(可接手)，逐格 pass/partial/fail。
H/V/A/E 黄金三角测尺；综合取木桶 min。（打分器自动：公式族 × 规则族取更严 verdict。）

## 步骤 7 · 诚实距 100 分清单 + 定下一战

逐格列出 verdict≠pass 的门 + 低分叶 = 距 100 分真实缺口清单，头挂「本图无金卡」横幅。
这份清单就是验收 checklist。短板层定位“下一战”（该补 IPO 信息工艺，还是 O/S/A 决策成熟度）。

## 反模式（别踩）

- 给 O、S、A 分别复制独立信息对象，造成来源、时间和分数漂移。
- 跳层：O 没想透就定 S、技术原型直接宣称商业价值、少于三周期宣称增长稳定、无回执宣称 A3。
- 虚报满分 / 用公式⚪冒充真据🟢 / 明细 PII 出仓 / 焊死"日更在线"式不会过期的假绿句。
